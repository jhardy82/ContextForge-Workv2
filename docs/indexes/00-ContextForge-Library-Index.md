## üîë Key Concepts Cross-Reference

### Context Ontology Framework (COF)

**13 Dimensions**:
1. Motivational - Why this work matters
2. Relational - Dependencies & connections
3. Situational - Environment & constraints
4. Resource - Time, skill, tools
5. Narrative - User/stakeholder journey
6. Recursive - Meta-patterns & processes
7. Computational - Algorithmic efficiency
8. Emergent - Unexpected interactions
9. Temporal - Timing & sequencing
10. Spatial - Topology & layout
11. Holistic - System-wide integration
12. Validation - Evidence & requirements
13. Integration - Fits back into whole

**Primary Source**: [03-Context-Ontology-Framework.md](03-Context-Ontology-Framework.md)
**Applied in**: All task schemas, context operations, quality gates

---

### Universal Context Law (UCL)

**Core Law**: *"No orphaned, cyclical, or incomplete context may persist in the system."*

**Enforcement**:
- Triple-Check Protocol (Build ‚Üí Logs ‚Üí Evidence)
- Strategic Session Audits (3/6/9 cadence)
- Compliance Gates (linkage, evidence, geometry, closure)

**Primary Source**: [03-Context-Ontology-Framework.md](03-Context-Ontology-Framework.md)
**Applied in**: [09-Development-Guidelines](09-Development-Guidelines.md), [13-Testing-Validation](13-Testing-Validation.md)

---

### Sacred Geometry Patterns

**Five Patterns**:

1. **Triangle (Stability)** - Three-layer architecture, Triple-Check Protocol
2. **Circle (Completeness)** - COF 13D analysis, Evidence bundles close the loop
3. **Spiral (Iteration)** - DuckDB velocity tracking, 2-week retrospectives
4. **Golden Ratio (Balance)** - Right-sized solutions (35% focused coverage, not 100%)
5. **Fractal (Modularity)** - Repository pattern at all layers

**Primary Source**: [03-Context-Ontology-Framework.md](03-Context-Ontology-Framework.md)
**Applied in**: [02-Architecture](02-Architecture.md), [09-Development-Guidelines](09-Development-Guidelines.md)

---

### UTMW Workflow

**5 Phases**:
1. **Understand** - Read code, review docs, trace execution
2. **Trust** - Run tests, validate logs, confirm assumptions
3. **Measure** - Collect metrics, establish baselines, identify gaps
4. **Validate** - Quality gates, compliance checks, evidence bundles
5. **Work** - Implement changes with full context

**Primary Source**: [13-Testing-Validation.md](13-Testing-Validation.md)
**Applied in**: All development workflows, code reviews, test design

---

### Database Authority Principle

**Core Principle** (Codex Addendum A):
- PostgreSQL (`172.25.14.122:5432/taskman_v2`) is **primary task management authority** for TaskMan-v2
- SQLite (`db/trackers.sqlite`) maintains legacy tracker data and supplementary context
- Runtime CSV mutation **blocked** (`direct_csv_access_blocked`)
- Legacy CSV constants only for migration

**Primary Source**: [02-Architecture](02-Architecture.md)
**Applied in**: [09-Development-Guidelines](09-Development-Guidelines.md), [04-Desktop-Application-Architecture](04-Desktop-Application-Architecture.md), all tracker operations

---

### Evidence Bundles

**Implementation**:
- Every state mutation produces structured log event
- SHA-256 cryptographic hash for tamper detection
- Append-only JSONL format
- 90% logging coverage target (Codex Addendum C)

**Primary Source**: [02-Architecture](02-Architecture.md)
**Applied in**: [09-Development-Guidelines](09-Development-Guidelines.md), [13-Testing-Validation](13-Testing-Validation.md)
